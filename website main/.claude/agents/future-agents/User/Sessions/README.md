# User Agent Sessions

This directory contains records of sessions where the User agent has been activated to perform exploratory testing, usability evaluation, defect identification, and experience optimization activities.

## Session Structure

Each session is organized in its own directory with the following structure:

```
Sessions/
  ├── SessionName/
  │     ├── README.md             # Session summary, objectives, and outcomes
  │     ├── metadata.json         # Session metadata (date, participants, etc.)
  │     ├── EXPLORATION_NOTES.md  # Detailed exploratory testing observations
  │     ├── USABILITY_FINDINGS.md # Usability issues and insights discovered
  │     ├── DEFECT_REPORT.md      # Documented bugs and unexpected behaviors
  │     └── RECOMMENDATIONS.md    # Suggested improvements and enhancements
  └── ...
```

## Session Templates

When creating a new User agent session, copy the appropriate template based on the session focus:

1. **Exploratory Testing**: Focus on discovering unexpected behaviors
2. **Usability Evaluation**: Focus on assessing user experience
3. **Bug Validation**: Focus on verifying reported issues
4. **Experience Enhancement**: Focus on improving existing experiences

## Session Naming Convention

Session directories follow this naming convention:
```
<ApplicationName>-<FocusArea>-<Date>
```

Example: `PaymentPortal-CheckoutFlow-2025-05-18`
