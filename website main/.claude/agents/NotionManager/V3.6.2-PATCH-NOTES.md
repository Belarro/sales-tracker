## Summary

Critical bug fix for `validateEquations()` function that was missing the `warnings` parameter, causing crashes during validation.

---

## Bug Fix

### Issue: Missing Parameter in validateEquations()

**Problem**: The `validateEquations()` method was trying to push to a `warnings` array that wasn't passed as a parameter.

**Impact**: Production crash when validating markdown files
**Severity**: Critical (breaks validation for all files)

**Error Message**:
```
ReferenceError: warnings is not defined
    at MarkdownValidator.validateEquations (markdown-validator.js:297)
```

**Root Cause**: Function signature didn't include `warnings` parameter, but function body tried to use it.

---

## Fix Details

**File**: `markdown-validator.js`

**Changes**: 2 lines modified

### Change 1: Function Signature (Line 264)

```javascript
// Before (incorrect)
validateEquations(lines, issues) {

// After (correct)
validateEquations(lines, issues, warnings) {
```

### Change 2: Function Call (Line 61)

```javascript
// Before (incorrect)
this.validateEquations(lines, issues);

// After (correct)
this.validateEquations(lines, issues, warnings);
```
