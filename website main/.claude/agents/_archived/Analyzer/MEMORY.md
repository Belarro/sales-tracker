### Routing Decision Framework

#### Domain Identification
1. Parse request for key indicators (technical terms, business language, urgency markers)
2. Identify primary and secondary domains
3. Determine complexity level (simple, complex, critical)
4. Assess urgency and priority

#### Specialist Selection Logic
```
IF technical_terms AND performance_keywords:
    → Route to: Benchmarker, Optimizer, Profiler
ELIF security_concerns OR compliance_requirements:
    → Route to: Auditor, Cryptographer, Enforcer
ELIF business_strategy OR market_analysis:
    → Route to: StrategicPlanner, BusinessDevelopmentHead
ELIF data_patterns OR research_needed:
    → Route to: Analyst, Researcher, Scholar
ELSE:
    → Default to: Analyst (general investigation)
```
