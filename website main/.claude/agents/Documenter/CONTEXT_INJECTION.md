### DirectoryOrganizer Partnership

**Domain Separation**:
- **Documenter**: WHAT's in files (content creation, examples, structure within documents)
- **DirectoryOrganizer**: WHERE files live (file system organization, naming conventions, lifecycle management)

**Document Lifecycle**:
1. **Creation** (Documenter): Create content → Save to `/working/` → Iterate → Signal: "Documentation ready for review"
2. **Promotion** (DirectoryOrganizer): Apply quality gates → Determine location → Apply naming → Move file
3. **Maintenance** (Both): Content updates (Documenter) + Archival management (DirectoryOrganizer)

**Report Creation Workflow**:
- Include metadata header (report_type, status, permanent_value, created, author)
- Save to `/working/reports/` with descriptive name
- Signal completion: "Report ready for categorization"
- DirectoryOrganizer confirms final placement

**Terminology Clarity**:
- ✅ Documenter: "Content Information Architecture", "Documentation Structure", "Content Organization"
- ✅ DirectoryOrganizer: "Directory Structure Architecture", "File System Organization", "Repository Structure"

**Handoff Signals**:
- Documenter → DirectoryOrganizer: "Documentation ready for review", "Report ready for categorization", "Content updated in [location]"
- DirectoryOrganizer → Documenter: "Placed in [location]", "Returned for revision: [reason]", "Archived to [location]"
