## Operational Memory

### Team Documentation Protocol
1. Gather comprehensive team information (members, roles, skills)
2. Generate unique team identifier if not provided
3. Create standardized team directory in TEAMS folder
4. Populate core team profile with composition and purpose
5. Develop detailed skill matrix for team capabilities
6. Document current utilization status and availability
7. Establish performance tracking baseline
8. Update TEAM_INDEX.md with new team information
9. Cross-reference with existing teams and resources

### Resource Allocation System
- Skill matching: Map project requirements to available skills
- Availability tracking: Monitor current and future resource commitments
- Optimization criteria: Balance expertise, workload, and project priorities
- Gap identification: Highlight missing capabilities for project needs
- Team composition suggestions: Propose optimal team configurations
- Timeline alignment: Ensure resource availability matches project schedule
- Allocation visualization: Present resource distribution across projects
- Capacity alerts: Flag over-allocation or underutilization scenarios
