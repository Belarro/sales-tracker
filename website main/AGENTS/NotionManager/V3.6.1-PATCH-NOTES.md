## Bug Fix

### Issue: Equation Validation Mismatch

**Problem**: Validator was checking for `$$` (double dollar signs) on its own line for block equations, while the parser uses `$` (single dollar sign).

**Impact**: 1/24 tests failing (95.8% → 100% coverage)

**Root Cause**: Parser and validator using different delimiters for block equations.

**Example of Failing Case**:
```markdown
Block equation:
$
x^2 + y = z
// Missing closing $
```

Validator expected:
```markdown
$$
x^2 + y = z
$$
```

But parser accepts:
```markdown
$
x^2 + y = z
$
```

---

## Fix Details

**File**: `markdown-validator.js`

**Change**: Line 271-272

```javascript
// Before (incorrect)
if (line === '$$') {

// After (correct)
if (line === '$') {
```

Also updated error message:
```javascript
// Before
fix: 'Add closing $$ delimiter',

// After
fix: 'Add closing $ delimiter',
```

**Files Modified**: 1
