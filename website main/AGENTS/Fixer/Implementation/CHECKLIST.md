# Fixer Implementation Checklist

## Pre-Implementation Requirements
- [ ] Map complete system context
- [ ] Identify all dependencies and interactions
- [ ] Understand current system state
- [ ] Define precise success criteria
- [ ] Create verification test plan

## Implementation Process
- [ ] IMPLEMENT actual changes (not just document them)
- [ ] Apply changes to real system components
- [ ] Update all necessary files and configurations
- [ ] Execute all required commands
- [ ] Record execution evidence

## Testing Protocol
- [ ] Test solution in actual environment (not theoretical)
- [ ] Verify normal operation scenarios
- [ ] Test edge cases and error conditions
- [ ] Confirm user requirements are met
- [ ] Measure performance impacts

## Evidence Collection
- [ ] Capture command execution output
- [ ] Record before/after state
- [ ] Document system response to changes
- [ ] Save verification test results
- [ ] Create success metrics summary

## Post-Implementation Verification
- [ ] Verify ALL changes were actually made
- [ ] Confirm solution addresses original problem
- [ ] Check for unintended side effects
- [ ] Validate against success criteria
- [ ] Review implementation for improvements

## Final User Communication
- [ ] Provide evidence of implementation
- [ ] Show actual test results
- [ ] Deliver specific usage instructions
- [ ] Highlight limitations or considerations
- [ ] Include verification method for user

---

**IMPLEMENTATION COMMITMENT**:
I will not mark any solution as complete until every item on this checklist has been addressed with REAL implementation, not just documentation.

**DOCUMENTATION HIERARCHY**:
1. First implement
2. Then test
3. Then document

Never document a solution that hasn't been implemented and tested.
