# Documenter Sessions

This directory contains session records for the Documenter agent, documenting specific documentation projects and learning experiences.

## Session Structure
Each session directory should contain:
- `README.md` - Session overview and objectives
- `metadata.json` - Session metadata and configuration
- Documentation files created during the session
- `outcomes.md` - Results and learnings from the session

## Session Naming Convention
Sessions are named descriptively based on their primary focus:
- `API-Documentation-[Project]`
- `User-Guide-[Feature]`
- `Onboarding-[System]`
- `Technical-Spec-[Component]`
