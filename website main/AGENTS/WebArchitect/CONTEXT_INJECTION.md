## Critical Architecture Patterns

### SSR vs CSR Decision Framework
- **SSR**: SEO-critical, content-heavy, initial load performance priority
- **CSR**: Interactive apps, authenticated experiences, real-time updates
- **Hybrid**: Next.js/Nuxt patterns, selective hydration, island architecture

### State Management Selection
- **Local**: useState/reactive for component-level
- **Shared**: Context/provide-inject for feature modules
- **Global**: Redux/Pinia for complex cross-cutting state
- **Server**: React Query/Apollo for remote data

### API Integration Patterns
- **REST**: Resource-based, HTTP caching, simple CRUD
- **GraphQL**: Complex data requirements, multiple sources, client-driven queries
- **Real-time**: WebSockets for live updates, SSE for server push

## Critical Lessons Learned

### Performance
- **Premature optimization wastes time**: Measure first, optimize bottlenecks only
- **Bundle size kills mobile**: Code splitting and lazy loading non-negotiable for production
- **Images dominate LCP**: Format optimization (WebP/AVIF) and responsive loading critical

### Architecture
- **Over-engineering reduces velocity**: Choose simplest solution meeting requirements
- **Framework lock-in is real**: Plan migration paths, avoid proprietary patterns
- **Team expertise trumps "best" tech**: Success requires team can execute

### Development Process
- **Performance budgets prevent drift**: Set metrics early, enforce in CI/CD
- **Accessibility from start saves time**: Retrofitting WCAG compliance is expensive
- **TypeScript strictness catches bugs**: Type safety investment pays off in maintenance
