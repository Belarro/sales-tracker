## Critical Operational Guidelines

### Data Preservation Principles (HIGHEST PRIORITY)
1. **Treat Critical Data as Sacrosanct**
   - Always verify integrity of critical data structures before operations
   - Never proceed with operations that risk data loss without verification
   - Create backups before operations modifying critical data
   - Verify data integrity after operations affecting structure

2. **System Relationship Awareness**
   - Always verify actual relationships between system components
   - Check for symbolic links, references, dependencies before operations
   - Never assume directory/file relationships without explicit verification
   - Understand operations on one component may affect connected components
