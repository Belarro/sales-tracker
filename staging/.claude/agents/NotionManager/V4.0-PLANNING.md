**Planning Date**: October 24, 2025
**Status**: Planning Phase

---

## 🎯 Vision

Transform NotionManager from a **one-way sync tool** (Git → Notion) into a **full-featured bidirectional system** that supports:

1. **Notion → Git sync** (pull changes from Notion back to Git)
2. **File attachments** (upload images, PDFs, etc.)
3. **Database operations** (create/query Notion databases)
4. **Real-time updates** (live sync with webhooks)

**Goal**: Make NotionManager the definitive Notion ↔ Git bridge for documentation workflows.

**Goal**: Pull Notion changes back to Git markdown files

### Use Cases

2. **Collaborative editing**

### Technical Requirements

#### 1. Notion Block → Markdown Converter
**Inverse of current parser** (markdown → blocks)

**Complexity**: High - Must handle all 38 current features in reverse

```javascript
class NotionToMarkdownConverter {
  // Convert Notion blocks to markdown
  async blockToMarkdown(block) {
    switch (block.type) {
      case 'heading_1': return `# ${this.richTextToMarkdown(block.heading_1.rich_text)}`;
      case 'heading_2': return `## ${this.richTextToMarkdown(block.heading_2.rich_text)}`;
      case 'paragraph': return this.richTextToMarkdown(block.paragraph.rich_text);
      case 'bulleted_list_item': return `- ${this.richTextToMarkdown(block.bulleted_list_item.rich_text)}`;
      case 'numbered_list_item': return `1. ${this.richTextToMarkdown(block.numbered_list_item.rich_text)}`;
      case 'code': return `\`\`\`${block.code.language}\n${block.code.rich_text[0].plain_text}\n\`\`\``;
      case 'image': return `![${block.image.caption}](${block.image.file.url})`;
      case 'table': return this.tableToMarkdown(block.table);
      case 'equation': return `$$${block.equation.expression}$$`;
      // ... handle all 14 block types
    }
  }

  // Convert Notion rich text to markdown
  richTextToMarkdown(richTextArray) {
    return richTextArray.map(rt => {
      let text = rt.plain_text;
      if (rt.annotations.bold) text = `**${text}**`;
      if (rt.annotations.italic) text = `*${text}*`;
      if (rt.annotations.code) text = `\`${text}\``;
      if (rt.annotations.strikethrough) text = `~~${text}~~`;
      if (rt.href) text = `[${text}](${rt.href})`;
      // ... handle all 24 formats
      return text;
    }).join('');
  }
}
```

**Challenges**:
- Notion's rich text is array-based, markdown is string-based
- Nested formatting edge cases
- Table cell formatting preservation
- Image URLs (Notion expiring URLs vs local paths)
- Page links (Notion IDs → Wiki-style links)

#### 2. Change Detection System
**Track when Notion pages are modified**

**Approach A: Polling** (Simpler)
```javascript
class NotionChangeDetector {
  async detectChanges(pageId, lastSyncTime) {
    const page = await this.notion.pages.retrieve({ page_id: pageId });
    const lastEdited = new Date(page.last_edited_time);

    if (lastEdited > lastSyncTime) {
      return {
        changed: true,
        lastEdited,
        editor: page.last_edited_by,
      };
    }
    return { changed: false };
  }

  async pollAllPages(interval = 60000) {
    setInterval(async () => {
      for (const [pageId, metadata] of this.trackedPages) {
        const changes = await this.detectChanges(pageId, metadata.lastSync);
        if (changes.changed) {
          await this.syncNotionToGit(pageId);
        }
      }
    }, interval);
  }
}
```

**Approach B: Webhooks** (Better but complex - v4.3)
- Requires server infrastructure
- Real-time updates
- Lower API usage

**Recommendation**: Start with Approach A (polling), move to webhooks in v4.3

#### 3. Git Commit Integration
**Auto-commit Notion changes with proper attribution**

```javascript
class NotionGitSync {
  async syncNotionToGit(pageId) {
    // 1. Fetch Notion page
    const blocks = await this.fetchAllBlocks(pageId);

    // 2. Convert to markdown
    const markdown = await this.converter.blocksToMarkdown(blocks);

    // 3. Determine file path
    const filePath = this.getFilePath(pageId); // From mapping

    // 4. Write to file
    await fs.writeFile(filePath, markdown, 'utf8');

    // 5. Git commit
    const editor = await this.getPageEditor(pageId);
    await this.gitCommit(filePath, {
      message: `docs: Update ${path.basename(filePath)} from Notion`,
      author: `${editor.name} (via Notion)`,
      coAuthor: 'NotionManager <noreply@notion.so>',
    });
  }

  async gitCommit(filePath, options) {
    const { message, author, coAuthor } = options;
    await exec(`git add "${filePath}"`);
    await exec(`git commit -m "${message}" --author="${author}" ${coAuthor ? `--trailer "Co-Authored-By: ${coAuthor}"` : ''}`);
  }
}
```

#### 4. Conflict Resolution
**Handle Git vs Notion conflicts**

**Scenarios**:
1. Both Git and Notion modified (conflict)
2. Git modified, Notion not (Git wins)
3. Notion modified, Git not (Notion wins)
4. Neither modified (no action)

**Strategy**:
```javascript
class ConflictResolver {
  async resolve(pageId, filePath) {
    const gitMtime = await this.getGitModifiedTime(filePath);
    const notionMtime = await this.getNotionModifiedTime(pageId);
    const lastSync = this.getLastSyncTime(pageId);

    const gitChanged = gitMtime > lastSync;
    const notionChanged = notionMtime > lastSync;

    if (gitChanged && notionChanged) {
      // CONFLICT
      return this.handleConflict(pageId, filePath);
    } else if (gitChanged) {
      // Git → Notion (existing v3.6 functionality)
      return this.syncGitToNotion(filePath, pageId);
    } else if (notionChanged) {
      // Notion → Git (new v4.0 functionality)
      return this.syncNotionToGit(pageId, filePath);
    }
    // No changes
    return { action: 'none' };
  }

  async handleConflict(pageId, filePath) {
    // Option 1: Last-write-wins (simple but risky)
    // Option 2: Create conflict markers (like Git merge)
    // Option 3: Prompt user for resolution

    // Recommended: Option 3 with fallback to Option 1
    if (this.interactiveMode) {
      return this.promptUserResolution(pageId, filePath);
    } else {
      // Fallback: Notion wins (since it's the UI)
      console.warn(`Conflict detected for ${filePath}. Notion version will be used.`);
      return this.syncNotionToGit(pageId, filePath);
    }
  }
}
```

#### 1. File Upload Handler
```javascript
class NotionFileUploader {
  async uploadFile(localPath, pageId) {
    // 1. Read file
    const fileBuffer = await fs.readFile(localPath);
    const mimeType = mime.lookup(localPath);

    // 2. Upload to Notion
    // Note: Notion API doesn't support direct uploads
    // Must use external hosting or Notion's internal upload

    // Option A: Upload to Notion via external URL
    const externalUrl = await this.uploadToExternalHost(fileBuffer);

    // Option B: Use Notion's internal file upload (undocumented API)
    const notionUrl = await this.notionInternalUpload(fileBuffer, pageId);

    return notionUrl;
  }

  async processMarkdownImages(markdown, pageId) {
    // Find all local image references
    const imageRegex = /!\[([^\]]*)\]\(([^)]+)\)/g;
    let processedMarkdown = markdown;

    for (const match of markdown.matchAll(imageRegex)) {
      const [fullMatch, altText, localPath] = match;

      if (this.isLocalFile(localPath)) {
        // Upload and replace
        const notionUrl = await this.uploadFile(localPath, pageId);
        processedMarkdown = processedMarkdown.replace(
          fullMatch,
          `![${altText}](${notionUrl})`
        );
      }
    }

    return processedMarkdown;
  }
}
```

#### 2. External Hosting Integration
**Challenge**: Notion API requires external URLs for file uploads

**Options**:
1. **AWS S3** - Professional, reliable, costs money
2. **Imgur** - Free for images, rate limited
3. **GitHub** - Use as CDN for repo files
4. **User-provided hosting** - Config option

**Recommendation**: Support multiple backends, let user choose

```javascript
class FileHostingAdapter {
  constructor(config) {
    this.backend = config.fileHosting || 'github'; // 'github', 's3', 'imgur'
  }

  async upload(fileBuffer, filename, mimeType) {
    switch (this.backend) {
      case 'github':
        return this.uploadToGitHub(fileBuffer, filename);
      case 's3':
        return this.uploadToS3(fileBuffer, filename, mimeType);
      case 'imgur':
        return this.uploadToImgur(fileBuffer);
      default:
        throw new Error(`Unknown backend: ${this.backend}`);
    }
  }

  async uploadToGitHub(fileBuffer, filename) {
    // Use GitHub repo as CDN
    // Store in assets/ directory
    const path = `assets/${filename}`;
    const content = fileBuffer.toString('base64');

    await this.github.repos.createOrUpdateFileContents({
      owner: this.config.repoOwner,
      repo: this.config.repoName,
      path,
      message: `Upload asset: ${filename}`,
      content,
    });

    return `https://raw.githubusercontent.com/${this.config.repoOwner}/${this.config.repoName}/main/${path}`;
  }
}
```

class NotionDatabaseManager {
  async createDatabase(parentPageId, schema) {
    return await this.notion.databases.create({
      parent: { page_id: parentPageId },
      title: [{ text: { content: schema.title } }],
      properties: schema.properties,
    });
  }

  async addEntry(databaseId, data) {
    return await this.notion.pages.create({
      parent: { database_id: databaseId },
      properties: this.formatProperties(data),
    });
  }

  async query(databaseId, filters) {
    return await this.notion.databases.query({
      database_id: databaseId,
      filter: filters,
      sorts: [{ property: 'Date', direction: 'descending' }],
    });
  }
}
