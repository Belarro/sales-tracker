# Engineer Sessions Directory

This directory contains all Engineer agent sessions. Each session documents a specific implementation task or functionality development project undertaken by the Engineer agent.

## Session Structure

Each session should be organized in its own directory with the following recommended structure:

```
SessionName/
  ├── README.md            # Session overview, context, and summary
  ├── metadata.json        # Session metadata (date, participants, etc.)
  ├── implementation_plan.md  # Detailed implementation approach (optional)
  └── outcomes.md          # Results and learnings from the session (optional)
```

## Session Naming Conventions

Session directories should follow these naming guidelines:

1. Use descriptive names that clearly indicate the implementation focus
2. Use PascalCase for directory names
3. Avoid unnecessary prefixes or technical jargon
4. Focus on the functionality being implemented rather than technical details

Examples:
- `UserAuthenticationSystem`
- `DataExportFunctionality`
- `NotificationServiceImplementation`
- `SearchOptimizationFeature`

## Session Documentation Standards

Each session README.md should include:

1. Clear description of the implementation task
2. Requirements and acceptance criteria
3. Implementation approach summary
4. Key technical decisions made during implementation
5. Testing and validation approach
6. Outcomes and learnings
