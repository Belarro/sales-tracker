**Release Date**: October 24, 2025
**Status**: ✅ Core Implementation Complete

## 🎯 Overview

v4.1 adds comprehensive file attachment support to NotionManager, enabling:
- Upload local images/files to external hosting
- Download and re-host expiring Notion images
- Multiple hosting backends (GitHub, S3, Imgur)
- Automatic markdown URL processing

**Critical Problem Solved**: Notion-hosted images expire after ~1 hour, breaking round-trip sync.

---

## ✨ Features Implemented

### 1. File Hosting Adapters (`file-hosting-adapters.js`)

**Base Architecture**:
- Abstract `FileHostingAdapter` class
- Pluggable backend system
- Unique filename generation (SHA-256 hash)
- Automatic download capability

**GitHub Backend** (Default, Free):
- Uploads to repository assets/ directory
- Uses Octokit for GitHub API
- 100MB file limit
- Generates stable GitHub raw URLs
- Handles file updates (checks existing SHA)

**S3 Backend** (Optional):
- AWS S3 integration
- Public read ACL
- Configurable bucket/region
- Professional CDN delivery

**Imgur Backend** (Optional):
- Free image hosting
- Images only (enforced)
- 10MB limit
- Rate limit aware

### 2. File Upload Manager (`file-upload-manager.js`)

**Core Capabilities**:
- Local file detection and upload
- Markdown image processing
- Notion image download/re-hosting
- Upload caching (prevents duplicates)
- File size validation
- MIME type detection

**Markdown Processing**:
```javascript
// Detect and upload local images
const { markdown, uploadedFiles } = await manager.processMarkdownImages(
  markdownContent,
  '/path/to/base/dir'
);

// Download expiring Notion images
const { markdown, downloadedImages } = await manager.downloadNotionImages(
  notionMarkdown
);
```

**Smart Caching**:
- Caches uploaded URLs by file path
- Avoids duplicate uploads
- In-memory cache (per instance)
- Upload queue prevents concurrent duplicates

### 3. Demo & Documentation

**Demo Script** (`demo-v4.1-file-uploads.js`):
- Shows all features
- Works without credentials (API examples)
- Demonstrates 3 backends
- File type examples

## 🚀 Usage

### Basic Setup

```javascript
import { createFileUploadManager } from './file-upload-manager.js';

// GitHub backend (default)
const manager = createFileUploadManager({
  adapter: {
    backend: 'github',
    github: {
      token: process.env.GITHUB_TOKEN,
      owner: 'username',
      repo: 'my-repo',
      branch: 'main',
      assetPath: 'assets',
    },
  },
  verbose: true,
});
```

### Upload Local Images

```javascript
// Process markdown - uploads local images automatically
const result = await manager.processMarkdownImages(
  markdownWithLocalImages,
  __dirname
);

console.log(result.markdown); // Updated markdown with hosted URLs
console.log(result.uploadedFiles); // Array of uploaded files
```

### Download Notion Images

```javascript
// Download and re-host expiring Notion images
const result = await manager.downloadNotionImages(notionMarkdown);

console.log(result.markdown); // Updated with stable URLs
console.log(result.downloadedImages); // Array of downloaded images
```

### Direct File Upload

```javascript
// Upload a specific file
const url = await manager.uploadFile('./path/to/image.png');
console.log(`Uploaded: ${url}`);

// Cached - subsequent calls return instantly
const cachedUrl = await manager.uploadFile('./path/to/image.png');
```

## 🔮 Integration with BidirectionalSyncOrchestrator

### v4.1.1 - Integration Complete ✅

**Git → Notion** (bidirectional-sync-orchestrator.js:277-285):
```javascript
// Read markdown file
let markdown = await fs.readFile(mapping.filePath, 'utf8');

// Process local images if file uploads enabled (v4.1)
if (this.enableFileUploads && this.fileUpload) {
  const baseDir = path.dirname(mapping.filePath);
  const result = await this.fileUpload.processMarkdownImages(markdown, baseDir);
  markdown = result.markdown;
  if (result.uploadedFiles.length > 0) {
    this.log(`Uploaded ${result.uploadedFiles.length} local file(s) to hosting`);
  }
}

// Then convert to Notion blocks
const blocks = markdownToNotionBlocks(markdown, 100, new Map());
```

**Notion → Git** (bidirectional-sync-orchestrator.js:196-203):
```javascript
// Convert to markdown
let markdown = this.converter.blocksToMarkdown(blocks);

// Download expiring Notion images if enabled (v4.1)
if (this.downloadNotionImages && this.fileUpload) {
  const result = await this.fileUpload.downloadNotionImages(markdown);
  markdown = result.markdown;
  if (result.downloadedImages.length > 0) {
    this.log(`Downloaded and re-hosted ${result.downloadedImages.length} Notion image(s)`);
  }
}

// Then write to file
await fs.writeFile(mapping.filePath, markdown, 'utf8');
```

  downloadNotionImages: true,       // Download expiring images during Notion→Git

      token: process.env.GITHUB_TOKEN,

## 🚨 Known Limitations

### Current Version

1. **No Integration Yet**: Not integrated with BidirectionalSyncOrchestrator
   - Standalone components ready
   - Integration planned for v4.1.1

2. **No Retry Logic**: Upload failures don't retry
   - Could add in future update
   - Error messages are clear

3. **No Compression**: Large images uploaded as-is
   - Could add Sharp integration
   - GitHub 100MB limit is generous

### Backend-Specific

**GitHub**:
- API rate limits (5,000 requests/hour)
- 100MB file size limit
- Requires repository write access

**S3**:
- Costs money (storage + bandwidth)
- Requires AWS configuration
- Need to manage permissions

**Imgur**:
- Images only
- 10MB limit
- Free tier rate limits (1,250 uploads/day)

## 🔧 Troubleshooting

### "GitHub token is required"
**Solution**: Set GITHUB_TOKEN in .env

### "File too large: XMB"
**Solution**: Compress image or use S3 backend

### "S3 backend requires @aws-sdk/client-s3"
**Solution**: `npm install @aws-sdk/client-s3`

### "Imgur only supports images"
**Solution**: Use GitHub/S3 for PDFs/videos

### Images not uploading
**Check**:
1. Backend credentials configured?
2. File exists at path?
3. File size within limits?
4. Network connectivity OK?

## 🎓 Best Practices

### Production Usage

1. **Use GitHub for Free Tier**:
   - No costs
   - Generous limits
   - Version control bonus

2. **Cache Aggressively**:
   - Manager caches by default
   - Reuse manager instance

3. **Validate File Sizes**:
   - Check before upload
   - Warn users of large files

4. **Handle Errors Gracefully**:
   - Log failures
   - Continue with other files
   - Don't crash sync process

### Security

1. **Protect Tokens**:
   - Use environment variables
   - Never commit tokens to git
   - Rotate periodically

2. **S3 Permissions**:
   - Minimum required permissions
   - Use IAM roles when possible
   - Monitor access logs

3. **Public URLs**:
   - Uploaded files are public
   - Don't upload sensitive data
   - Consider S3 signed URLs for private files

## 📊 Performance Tips

1. **Batch Processing**: Upload multiple files concurrently
2. **Cache Reuse**: Keep manager instance alive
3. **Lazy Loading**: Only upload when needed
4. **Compression**: Optimize images before upload
