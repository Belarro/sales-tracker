## Technical Implementation

### Notion Table Block Structure

```javascript
{
  type: 'table',
  table: {
    table_width: 3,              // Number of columns
    has_column_header: true,     // First row is header
    has_row_header: false,       // First column is not header
    children: [                  // Array of table_row blocks
      {
        type: 'table_row',
        table_row: {
          cells: [                // Array of rich_text arrays (one per cell)
            [{ type: 'text', text: { content: 'Name' } }],
            [{ type: 'text', text: { content: 'Status' } }],
            // ...
          ]
        }
      },
      // More rows...
    ]
  }
}
```

### Parser Changes

**Before (v2.1)** - Code Block Approach:
```javascript
// Collected table as plain text
blocks.push({
  type: 'code',
  code: {
    rich_text: [{ type: 'text', text: { content: tableLines.join('\n') } }],
    language: 'plain text',
  },
});
```

**After (v2.2)** - Native Table Approach:
```javascript
// Parse each cell with rich text formatting
const tableChildren = dataRows.map(row => {
  const cells = [];
  for (let j = 0; j < tableWidth; j++) {
    const cellContent = row[j] || '';
    cells.push(parseRichText(cellContent)); // ⭐ Rich text parsing!
  }

  return {
    type: 'table_row',
    table_row: { cells: cells },
  };
});

blocks.push({
  type: 'table',
  table: {
    table_width: tableWidth,
    has_column_header: true,
    children: tableChildren,
  },
});
```

**Key difference**: Each cell goes through `parseRichText()`, so `<green>text</green>` becomes colored text, `**bold**` becomes bold, etc.

## Known Limitations

### Still Not Supported

1. **Nested tables**: Tables within table cells (Notion limitation)
2. **Merged cells**: Colspan/rowspan not supported (Notion limitation)
3. **Cell alignment**: Left/center/right alignment from markdown (Notion uses auto-alignment)
4. **Table width**: Cannot specify specific column widths (Notion auto-sizes)

### Workarounds

These limitations are from Notion's table block API, not our parser. Notion tables are simpler than HTML tables.
