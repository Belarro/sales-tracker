   ```javascript
   if (this.enableFileUploads && this.fileUpload) {

     const result = await this.fileUpload.processMarkdownImages(markdown, baseDir);
     markdown = result.markdown;

   ```

4. **Notion → Git sync integration** (lines 196-203):
   ```javascript
   // Download expiring Notion images if enabled (v4.1)
   if (this.downloadNotionImages && this.fileUpload) {
     const result = await this.fileUpload.downloadNotionImages(markdown);
     markdown = result.markdown;
     if (result.downloadedImages.length > 0) {
       this.log(`Downloaded and re-hosted ${result.downloadedImages.length} Notion image(s)`);
     }
   }
   ```

**Key Design Decisions**:
- Features are **optional** (disabled by default) for backward compatibility
- File processing happens at the right points in the sync workflow
- Logging provides visibility into upload/download activity
- Error handling is inherited from FileUploadManager

## 🔄 How It Works

### Git → Notion Sync with File Upload

1. User edits markdown file with local images:
   ```markdown
   ![Architecture](./diagrams/system.png)
   ```

2. User calls `orchestrator.syncGitToNotion(pageId)`

3. Orchestrator workflow:
   ```
   Read markdown from Git
        ↓
   [v4.1.1] Detect local images
        ↓
   [v4.1.1] Upload to GitHub/S3/Imgur
        ↓
   [v4.1.1] Replace with hosted URLs
        ↓
   Convert to Notion blocks
        ↓
   Update Notion page
   ```

4. Notion page now has permanent image URLs

### Notion → Git Sync with Image Download

1. User edits Notion page with images

2. User calls `orchestrator.syncNotionToGit(pageId)`

3. Orchestrator workflow:
   ```
   Fetch Notion blocks
        ↓
   Convert to markdown
        ↓
   [v4.1.1] Detect expiring Notion URLs
        ↓
   [v4.1.1] Download images
        ↓
   [v4.1.1] Re-upload to stable hosting
        ↓
   [v4.1.1] Replace with hosted URLs
        ↓
   Write markdown to Git
        ↓
   Create Git commit
   ```

4. Git markdown has permanent image URLs

## 🎓 Usage Examples

### Basic Setup

```javascript
import { createBidirectionalSyncOrchestrator } from './bidirectional-sync-orchestrator.js';

const orchestrator = await createBidirectionalSyncOrchestrator({
  notionToken: process.env.NOTION_TOKEN,

  // v4.1.1 - Enable file features
  enableFileUploads: true,          // Upload local images during Git→Notion
  downloadNotionImages: true,       // Download expiring images during Notion→Git

  fileHosting: {
    backend: 'github',              // or 's3', 'imgur'
    github: {
      token: process.env.GITHUB_TOKEN,
      owner: 'username',
      repo: 'my-docs',
      assetPath: 'assets',
    },
  },

  verbose: true,
});
```

### Sync Operations

```javascript
// Register page
await orchestrator.registerPage(
  '29660b6e-...',
  'docs/README.md',
  'Documentation'
);

// Sync Git → Notion (uploads local images automatically)
await orchestrator.syncGitToNotion('29660b6e-...');

// Sync Notion → Git (downloads expiring images automatically)
await orchestrator.syncNotionToGit('29660b6e-...');

// Continuous sync (all automatic)
await orchestrator.startContinuousSync({ autoSync: true });
```

### Disable File Features (Backward Compatible)

```javascript
// Without file features (v4.0 behavior)
const orchestrator = await createBidirectionalSyncOrchestrator({
  notionToken: process.env.NOTION_TOKEN,
  // enableFileUploads and downloadNotionImages default to false
});
```

## 📈 Performance Impact

**Minimal overhead when disabled**:
- `enableFileUploads: false` → No FileUploadManager created, zero overhead
- Features are opt-in, backward compatible

**When enabled**:
- File upload: ~1-2s per image (network dependent)
- Cached uploads: <1ms (instant)
- Image download: ~1s per image
- Markdown processing: <10ms (regex matching)

**Rate Limiting**:
- Notion API: Existing 600ms delays unchanged
- GitHub API: 5,000 requests/hour limit (generous)
- Upload queue prevents concurrent duplicate uploads
