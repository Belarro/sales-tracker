## 🔧 Technical Implementation

### Parser Enhancement (markdown-parser-enhanced.js:316-355)

**Added image block detection before table parsing**:
```javascript
// Image: ![alt text](url)
else if (line.match(/^!\[([^\]]*)\]\(([^)]+)\)/)) {
  const match = line.match(/^!\[([^\]]*)\]\(([^)]+)\)/);
  const imageUrl = match[2];

  // Check if next line is a caption
  let caption = [];
  if (i + 1 < lines.length) {
    const nextLine = lines[i + 1].trim();
    // Caption detection: starts with _ or is short plain text
    if (nextLine && (
      nextLine.startsWith('_') ||
      (nextLine.length < 100 && !nextLine.match(/^[#>*\-\d`!?|]/) && !nextLine.startsWith('```'))
    )) {
      const captionText = nextLine.startsWith('_') ? nextLine.substring(1).trim() : nextLine;
      caption = parseRichText(captionText);
      i++; // Skip caption line
    }
  }

  const imageBlock = {
    object: 'block',
    type: 'image',
    image: {
      type: 'external',
      external: { url: imageUrl },
    },
  };

  // Add caption if present
  if (caption.length > 0) {
    imageBlock.image.caption = caption;
  }

  blocks.push(imageBlock);
  blockCount++;
}
```

### Caption Detection Logic

**Two methods**:
1. **Explicit**: Line starts with `_` → Always treated as caption
2. **Auto-detect**: Plain text < 100 chars without markdown syntax

**Excluded from auto-detection**:
- Lines starting with: `#`, `>`, `*`, `-`, digits, `` ` ``, `!`, `?`, `|`
- Lines starting with ` ``` ` (code blocks)
- Lines > 100 characters (likely paragraphs, not captions)

**Why this works**:
- Most captions are short (< 100 chars)
- Captions rarely start with markdown syntax
- False positives are minimal in real documentation

### Rich Text in Captions

Captions go through `parseRichText()`, so they support:
- **Bold**: `**text**`
- *Italic*: `*text*`
- __Underline__: `__text__`
- `Code`: `` `text` ``
- ~~Strikethrough~~: `~~text~~`
- Colors: `<green>text</green>`
- Backgrounds: `<yellow_background>text</yellow_background>`
- Links: `[text](url)`

## 🔍 Known Limitations

### 1. Alt Text Not Preserved
**Issue**: Markdown alt text `![alt](url)` is parsed but not stored
**Reason**: Notion API doesn't support alt text on image blocks
**Workaround**: Use captions for accessibility text

### 2. File Uploads Not Supported
**Issue**: Only external URLs work: `![img](url)` ✅
**Limitation**: Local files don't work: `![img](./local.png)` ❌
**Reason**: Notion API requires multipart/form-data for uploads (complex)
**Timeline**: Planned for v3.5 or v4.0

### 3. Images in Tables - Experimental
**Issue**: Images in table cells may not render correctly
**Status**: Parser creates blocks, but Notion rendering is unpredictable
**Workaround**: Use images before/after tables instead

### 4. Caption Detection False Positives (Rare)
**Issue**: Very rare cases where short paragraph is treated as caption
**Mitigation**: Caption must be < 100 chars and no markdown syntax
**Workaround**: Add blank line between image and text to prevent

## 🎓 Best Practices

### 1. Use High-Quality Images
```markdown
✅ Good: ![diagram](https://example.com/hq-diagram.png?w=1600)
❌ Bad:  ![diagram](https://example.com/tiny-image.png?w=100)
```

### 2. Always Add Captions
```markdown
✅ Good: ![chart](url)\n_Monthly revenue trends_
❌ Bad:  ![chart](url)
```

### 3. Use Descriptive Alt Text
```markdown
✅ Good: ![OAuth 2.0 sequence diagram](url)
❌ Bad:  ![image1](url)
```

### 4. External URLs Only
```markdown
✅ Good: ![img](https://example.com/image.png)
❌ Bad:  ![img](./local/image.png)
```

### 5. Keep Captions Concise
```markdown
✅ Good: _System architecture showing microservices and databases_
❌ Bad:  _This is a very detailed caption that goes on and on explaining every single detail of the image including all the components and their relationships and how they interact with each other..._
```
