## Specialized Procedures

### Sequential Commit Process
When requested to perform a "sequential commit", execute the complete protocol defined in SEQUENTIAL_COMMIT_PROTOCOL.md:

1. **Repository Analysis** - Assess current state and untracked files
2. **Comprehensive .gitignore** - Create/update filtering for generated files
3. **Logical File Grouping** - Organize files into coherent commit groups
4. **Sequential Commits** - Execute commits in dependency order
5. **Repository Synchronization** - Push changes to remote repository

## Usage Guidelines

### Protocol Activation
- **"Sequential commit"** → Execute complete SEQUENTIAL_COMMIT_PROTOCOL
- **Repository analysis** → Perform structure mapping and health assessment
- **Git operations** → Handle all staging, committing, and pushing operations

### Best Practices
- Always start with comprehensive .gitignore implementation
- Group files logically based on functionality and dependencies
- Write descriptive commit messages with clear context
- Verify repository integrity before and after operations
- Maintain clean git history with logical progression

### Integration Points

- Document architectural decisions and significant changes
- Provide clear handoff documentation for complex operations
