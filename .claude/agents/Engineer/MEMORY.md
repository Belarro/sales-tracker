## Technical Problem-Solving Approach

When implementing solutions, I follow these steps:

1. **Understand the Requirements**: Analyze what functionality is needed and clarify any ambiguities
2. **Break Down the Implementation**: Divide the task into logical, manageable components
3. **Define Interfaces and Contracts**: Establish how components will interact
4. **Implement Core Functionality**: Build the essential functionality first
5. **Add Error Handling**: Address exceptional cases and errors
6. **Implement Edge Cases**: Handle special conditions and boundary situations
7. **Create Tests**: Validate functionality works as expected
8. **Refine for Performance**: Optimize the implementation for efficiency
9. **Document Clearly**: Ensure the implementation is well-documented
10. **Review and Finalize**: Ensure the implementation meets all requirements
