## 🎯 Objective

Add file attachment support to NotionManager, enabling:
1. Upload local images/files to external hosting
2. Handle Notion's expiring image URLs
3. Support multiple hosting backends (GitHub, S3, Imgur)
4. Automatic URL replacement in markdown

---

## 💡 Motivation

### Critical Issue from v4.0 Research
**Finding**: Notion-hosted images expire after ~1 hour
**Impact**: Round-trip sync loses images
**Solution**: Download and re-upload to stable hosting

### User Value
- **Embed local images** in markdown → auto-upload to Notion
- **Preserve images** during Notion → Git sync
- **Stable URLs** that don't expire
- **Multiple backends** for flexibility

## 🏗️ Architecture

### Core Components

#### 1. FileHostingAdapter (Abstract Base)
```javascript
class FileHostingAdapter {
  async upload(fileBuffer, filename, mimeType) {}
  async download(url) {}
  getUrl(filename) {}
}
```

#### 2. GitHubHostingAdapter (Default)
```javascript
class GitHubHostingAdapter extends FileHostingAdapter {
  constructor(config) {
    this.owner = config.repoOwner;
    this.repo = config.repoName;
    this.branch = config.branch || 'main';
    this.assetPath = config.assetPath || 'assets';
  }

  async upload(fileBuffer, filename, mimeType) {
    // Use GitHub API to commit file to assets/ directory
    const path = `${this.assetPath}/${filename}`;
    const content = fileBuffer.toString('base64');

    await octokit.repos.createOrUpdateFileContents({
      owner: this.owner,
      repo: this.repo,
      path,
      message: `Upload asset: ${filename}`,
      content,
      branch: this.branch,
    });

    return `https://raw.githubusercontent.com/${this.owner}/${this.repo}/${this.branch}/${path}`;
  }
}
```

#### 3. S3HostingAdapter
```javascript
class S3HostingAdapter extends FileHostingAdapter {
  constructor(config) {
    this.bucket = config.bucket;
    this.region = config.region;
    this.s3 = new S3Client({ region: this.region });
  }

  async upload(fileBuffer, filename, mimeType) {
    await this.s3.putObject({
      Bucket: this.bucket,
      Key: filename,
      Body: fileBuffer,
      ContentType: mimeType,
      ACL: 'public-read',
    });

    return `https://${this.bucket}.s3.${this.region}.amazonaws.com/${filename}`;
  }
}
```

#### 4. ImgurHostingAdapter (Images Only)
```javascript
class ImgurHostingAdapter extends FileHostingAdapter {
  constructor(config) {
    this.clientId = config.clientId;
  }

  async upload(fileBuffer, filename, mimeType) {
    // Only supports images
    if (!mimeType.startsWith('image/')) {
      throw new Error('Imgur only supports images');
    }

    const response = await fetch('https://api.imgur.com/3/image', {
      method: 'POST',
      headers: {
        Authorization: `Client-ID ${this.clientId}`,
      },
      body: fileBuffer.toString('base64'),
    });

    const data = await response.json();
    return data.data.link;
  }
}
```

#### 5. FileUploadManager
```javascript
class FileUploadManager {
  constructor(adapter) {
    this.adapter = adapter; // FileHostingAdapter instance
    this.cache = new Map(); // filename → URL cache
  }

  async uploadFile(localPath) {
    // 1. Check cache
    if (this.cache.has(localPath)) {
      return this.cache.get(localPath);
    }

    // 2. Read file
    const fileBuffer = await fs.readFile(localPath);
    const mimeType = mime.lookup(localPath);
    const filename = path.basename(localPath);

    // 3. Upload
    const url = await this.adapter.upload(fileBuffer, filename, mimeType);

    // 4. Cache
    this.cache.set(localPath, url);

    return url;
  }

  async processMarkdownImages(markdown, baseDir) {
    // Find all local image references
    const imageRegex = /!\[([^\]]*)\]\(([^)]+)\)/g;
    let processedMarkdown = markdown;

    for (const match of markdown.matchAll(imageRegex)) {
      const [fullMatch, altText, imagePath] = match;

      // Only process local files (not URLs)
      if (!imagePath.startsWith('http://') && !imagePath.startsWith('https://')) {
        const absolutePath = path.resolve(baseDir, imagePath);

        try {
          const hostedUrl = await this.uploadFile(absolutePath);
          processedMarkdown = processedMarkdown.replace(
            fullMatch,
            `![${altText}](${hostedUrl})`
          );
        } catch (error) {
          console.warn(`Failed to upload ${imagePath}: ${error.message}`);
        }
      }
    }

    return processedMarkdown;
  }

  async downloadNotionImage(notionUrl) {
    // Download expiring Notion image
    const response = await fetch(notionUrl);
    const buffer = await response.buffer();

    // Extract filename from URL or generate
    const filename = this.extractFilename(notionUrl) || `notion-${Date.now()}.png`;

    // Upload to stable hosting
    return await this.uploadFile(buffer, filename);
  }
}
```

#### 6. Integration with v4.0

**In BidirectionalSyncOrchestrator**:
```javascript
class BidirectionalSyncOrchestrator {
  constructor(options) {
    // ...existing code...

    // Add file upload manager
    this.fileUpload = new FileUploadManager(
      this.createHostingAdapter(options.fileHosting)
    );
  }

  createHostingAdapter(config) {
    const backend = config?.backend || 'github';

    switch (backend) {
      case 'github':
        return new GitHubHostingAdapter(config.github);
      case 's3':
        return new S3HostingAdapter(config.s3);
      case 'imgur':
        return new ImgurHostingAdapter(config.imgur);
      default:
        throw new Error(`Unknown backend: ${backend}`);
    }
  }

  async syncGitToNotion(pageId, options = {}) {
    // ...existing code...

    // NEW: Process images before converting to Notion
    const processedMarkdown = await this.fileUpload.processMarkdownImages(
      markdown,
      path.dirname(mapping.filePath)
    );

    // Convert processedMarkdown instead of original
    const blocks = markdownToNotionBlocks(processedMarkdown, 100, new Map());

    // ...rest of existing code...
  }

  async syncNotionToGit(pageId, options = {}) {
    // ...existing code...

    // NEW: Download expiring images
    if (options.downloadImages) {
      markdown = await this.downloadNotionImages(markdown);
    }

    // ...rest of existing code...
  }

  async downloadNotionImages(markdown) {
    const imageRegex = /!\[([^\]]*)\]\((https:\/\/[^)]+notion[^)]+)\)/g;

    for (const match of markdown.matchAll(imageRegex)) {
      const [fullMatch, altText, notionUrl] = match;

      try {
        const stableUrl = await this.fileUpload.downloadNotionImage(notionUrl);
        markdown = markdown.replace(fullMatch, `![${altText}](${stableUrl})`);
      } catch (error) {
        console.warn(`Failed to download ${notionUrl}: ${error.message}`);
      }
    }

    return markdown;
  }
}
```

## 📋 File Type Support

### Images
- **Formats**: PNG, JPG, GIF, SVG, WebP
- **Notion block**: `image` block with external URL
- **All backends**: Supported

### PDFs
- **Format**: PDF
- **Notion block**: `pdf` block or `file` block
- **GitHub/S3**: Supported
- **Imgur**: Not supported

### Videos
- **Formats**: MP4, MOV, AVI
- **Notion block**: `video` block with external URL
- **GitHub/S3**: Supported (large files)
- **Imgur**: Not supported

### Other Files
- **Formats**: ZIP, TXT, CSV, etc.
- **Notion block**: `file` block with download link
- **GitHub/S3**: Supported
- **Imgur**: Not supported

## 🚨 Risks & Mitigations

### Risk 1: Large File Sizes
**Issue**: GitHub/S3 have size limits
**Mitigation**:
- Check file size before upload
- Warn if >10MB (GitHub limit: 100MB)
- Compress images if needed

### Risk 2: API Rate Limits
**Issue**: GitHub API has rate limits
**Mitigation**:
- Cache uploaded files
- Batch uploads if possible
- Respect rate limits (check headers)

### Risk 3: Cost (S3)
**Issue**: S3 storage and bandwidth cost money
**Mitigation**:
- Default to GitHub (free)
- Warn user about costs
- Provide cost calculator

### Risk 4: Imgur Rate Limits
**Issue**: Imgur free tier has limits
**Mitigation**:
- Check rate limit before upload
- Fallback to GitHub if limit reached
- Cache aggressively
