## 📖 Complete Syntax Guide

### 1. Images

```markdown
![Architecture](https://example.com/diagram.png)
_System architecture showing all components_
```

- **Formats**: PNG, JPEG, GIF, SVG, WEBP
- **Captions**: Auto-detect or explicit with `_text_`
- **Formatted captions**: Supports bold, colors, links

### 2. Page Links

```markdown
Reference @NotionManager or [[Athena]] for details.
See [[Documentation]] and @Developer for more info.
```

- **@mentions**: `@PageName` (single word or CamelCase)
- **Wiki links**: `[[Page Name]]` (multi-word with brackets)
- **Smart resolution**: Automatic page ID lookup with caching
- **Graceful fallback**: Missing pages → plain text (no errors)

### 3. Math Equations

```markdown
Inline: Einstein's $$E = mc^2$$ is famous.

Block:
$$
x = \frac{-b \pm \sqrt{b^2-4ac}}{2a}
$$
```

- **Inline**: `$$expression$$` within text flow
- **Block**: `$$` on own line starts display equation
- **LaTeX/KaTeX**: Full support for mathematical notation
- **Features**: Fractions, Greek letters, matrices, integrals, limits, summations, etc.

## 💻 Implementation Details

### Parser Architecture (v3.5)

**File**: `markdown-parser-enhanced.js`

**Key Functions**:
1. `parseRichText(text)` - Basic formatting
2. `parseRichTextWithPageLinks(text, pageMap)` - With page mentions + equations
3. `markdownToNotionBlocks(markdown, maxBlocks, pageMap)` - Complete converter

**Pattern Matching**:
```javascript
// Enhanced pattern with all features
const pattern = /(\$\$([^\$]+)\$\$|\[\[([^\]]+)\]\]|@(\w+)|\[([^\]]+)\]\(([^)]+)\)|<([\w_]+)>([^<]+)<\/\7>|\*\*([^*]+)\*\*|__([^_]+)__|`([^`]+)`|~~([^~]+)~~|\*([^*]+)\*)/;
```

**Order of precedence** (critical):
1. Inline equations `$$..$$`
2. Wiki links `[[..]]`
3. @mentions
4. Regular links `[..](...)`
5. Colors `<..>..</..>`
6. Bold, underline, code, strikethrough, italic

### Page Link Resolution

**File**: `page-link-resolver.js`

**Class**: `PageLinkResolver`

**Features**:
- Smart caching (5-minute TTL)
- Fuzzy matching support
- Batch resolution
- Graceful error handling
- Case-insensitive lookup

**Methods**:
```javascript
async searchPage(title, exactMatch = false)
async resolveBatch(titles)
extractPageReferences(text)
clearCache()
getCacheStats()
```

### Equation Support

**Inline equations**:
- Pattern: `\$\$([^\$]+)\$\$`
- Creates `{ type: 'equation', equation: { expression: '...' } }`
- Renders within text flow

**Block equations**:
- Delimiter: `$$` on own line
- Collects lines until closing `$$`
- Creates equation block
- Renders centered and larger

## 🎓 Best Practices

### 1. Images
- ✅ Use high-quality external URLs
- ✅ Always add captions
- ✅ Use descriptive alt text
- ❌ Don't use local file paths (not supported)

### 2. Page Links
- ✅ Use @mentions for agents/pages
- ✅ Use [[wiki]] for multi-word titles
- ✅ Check links resolve before sharing
- ❌ Don't assume all pages exist (handle gracefully)

### 3. Equations
- ✅ Use inline for simple math in text
- ✅ Use block for complex/important equations
- ✅ Test LaTeX syntax before syncing
- ❌ Don't use unsupported LaTeX packages

### 4. Formatting
- ✅ Combine features: `**@NotionManager**` with `<green>$$x^2$$</green>`
- ✅ Keep captions concise (< 100 chars for auto-detect)
- ✅ Use consistent syntax throughout docs
- ❌ Don't nest incompatible formats
